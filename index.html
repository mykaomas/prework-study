Prework Study Guide

HTML

  • HTML is seen as the structure of a website like building the foundation of a house
  • The head element contains information about the webpage
  • The body element represents the visible content shown to user
  • H1-H6 represnets the headers and sizing. Using the H1 header once per page
  • HTML elements are case sensitive

CSS

  • CSS defines the style of the "house" ie windows, decorations, chimney
  • Using the style element defines the CSS style that will be associated with the website
  • When assigining colors we can use the hexadecimal code to pick from a wide range of colors
  • When adding external CSS code the code has to be linked back to the HTML code in order to have the changes visible
  • DRY aka Do Not Repeat Yourself groups elements with the same styling together to avoid repetition
  • A margin indicates how much space we want outside the element while padding indicates how much space we want inside the element around our content

Git

  • Git is a version constrol system that helps manage and keep track of each version of a project
  • Git allows you to track changes locally but also allows for the collaboration with others
  • Git commands always start with git
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Using Git commands allows us to create a repository or "repo" to view the different versions of our code

JavaScript

  • JavaScript provides fuctionality to the webpage which allows users to be interactive
  • JavaScript is read from top to bottom; order of code matters
  • Control flow is the order in which a computer executes code in a script
  • A variable is a named container that allows us to store data in our code
  • A function's name is always followed up with parentheses
  • Array is a single variable that holds a group of data
  • The location of the data item in an array is called an index.
  • Arrays are zero-indexed so the first item in an array starts with 0 not 1.
  • Defining a function does not mean it will execute in our code automatically
